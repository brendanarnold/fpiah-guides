A Skeleton Project - Windows Phone Development for People in a Hurry

Windows Phone SDK does not enforce a standard layout for the project. Here is a tried and tested layout.

    1. Converters.cs
        1. MyUserControl1.xaml.cs
        1. MyUserControl2.xaml.cs
      1. ProfilesPage.xaml.cs
      1. EditProfilePage.xaml.cs
      1. OrdersPage.xaml.cs
      1. EditOrderPage.xaml.cs
    1. Profile.cs
    2. Order.cs
    3. StorageInterface.cs
    4. StorageInitialiser.cs
    5. StorageMigrater.cs
    1. AppViewModel.cs
  7. AppConstants.cs
    1. App.xaml.cs

View

  • All files in folder are under the 'YourAppName.View' namespace
  • Contains all XAML pages and their code-behind files apart from App.xaml
  • UserControls folder contains user controls
  • Converters.cs is a files that contains all the Converter classes

Model

  • All files in folder are under the 'YourAppName.Model' namespace
  • Includes classes for the objects retrieved from your datasource e.g. Profile.cs and Order.cs
  • StorageInterface.cs is a class that implements the interface to the datasource e.g. database, webservice, XAML files (n.b. this is the ToDoDataContext.cs file in the Local database app with MVVM for Windows Phone tutorial)
  • StorageInitialiser.cs contains a class and objects that populate the database, XML storage etc. on first use with e.g. Order templates. This should contain code that is run once on a fresh install.
  • StorageMigrater.cs contains code for updating the database, XML storage between different versions of the app. This is run once when the app is updated.

View-Model

  • All files are under the 'YourAppName.ViewModel' namespace
  • AppViewModel.cs implements a single AppViewModel class which loads the necessary data form the Model into its properties. These properties are mirrored automatically in the View

SampleData

  • Expression Blend can create sample data. That data can go in the SampleData directory

Includes

  • A space for included libraries (.dll files)
  • Pretty much every Silverlight Windows phone project will need the Windows Phone Toolkit
  • Create a separate folder for each library

Images

  • A space for images that are included in the project
  • For all images set the followinf properties in Visual Studio so they are included in the built project,
    • Build Type: Content
    • Copy to Output Directory: Copy if newer

Root directory

  • Classes should be under the root namespace e.g. 'MyAppName'
  • AppConstants is a static class with 'const' properties that are used throughout the app
  • App.xaml creates an instance of the ViewModel class which can be referred to anywhere in the app as 'App.VM'
