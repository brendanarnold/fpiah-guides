Understanding Windows Phone Project files

When creating a new Project a series of file are created as follows,

    1. AppManifest.xml
    2. AssemblyInfo.cs
    3. WMAppManifest.xml
    1. App.xaml.cs
    1. MainPage.xaml.cs
  5. ApplicationIcon.png
  6. Background.png
  7. SplashScreenImage.png

Properties

  • Properties folder contains Windows Phone App metadata
  • Data in these files can be edited safely using the 'Project → YourAppName Properties' window in Visual Studio, otherwise these files can be mostly be left alone.
  • AppManifest.xml contains a list of extra resource files to be included in the final built .xap package
  • AssemblyInfo.cs contains metadata for management of the .NET .dll that is compiled from the project. Contains things like .dll version number
  • WMAppManifest.xml contains metadata that is important for the Windows Phone Store such as what permissions are required (i.e. does the app use GPS?) and the company name, version etc. Also contains the reference for the first page loaded in your app (i.e. MainPage.xaml)

References

  • Contains references to libraries, see adding libraries to a project

Other files

  • App.xaml is the first xaml file loaded by the project when it runs. It does not display anything but is where global XAML snippets are placed
  • App.xaml.cs properties on this class can be accessed in code under the global App object. It is a good place to attach the View-Model instance. Also provides the app 'Launching', 'Closing', 'Activated' and 'Deactivated' methods
  • MainPage.xaml is an ordinary Windows Phone XAML page. However it is referenced in WMAppManifest.xml to be the first page loaded when the app starts. When moving to the View folder, be sure to update the reference.
