Windows Phone Development for People in a Hurry

Overview

Architecture

  • There are four options for developing for Window Phone, of which we focus on the latter,
    • HTML/Javascript (introduced in Windows Phone 8)
    • Native C++ (introduced in Windows Phone 8)
    • XNA platform (for 3D applications)
    • Silverlight platform (for 2D applications)
  • The Silverlight Windows Phone platform is a subset of Silverlight which is itself a subset of the Windows Presentation Foundation. When searching for solutions first try including 'Windows Phone' then try 'Silverlight', then try 'WPF'
  • The UI for all of the above is written in 'eXtensible Application Markup Language' (XAML) which is a diaect of XML and the backend is written in either C# or Visual Basic (VB)
  • If you are coming with no knowledge of VB or C#, opt for C#
  • XAML/C# is directly comparable to HTML/Javascript however,
    • XAML/C# are much cleaner languages
    • XAML/C# are tailored for the task of writing applications
    • HTML/Javascript apps takes less changes to port to another platform
    • There is no direct equivelent for CSS although you can save XAML styles and apply them to multiple XAML elements

Solutions and Projects

  • A Windows Phone app is organised into a 'Solution' and 'Projects'
  • A 'Solution' groups together Projects that are associated with your application
  • Usually you just have one project in your solution for the app itself, however there may be other projects, for example,
    • unit tests
    • a desktop front-end
    • a web front-end
    • a separate library spun out of the main project
  • A Window Phone project generates lots of files automatically, see Understanding Windows Phone Project files for a guide on what they do
  • .cproj or .vbproj and .sln files are just XML. Take a look inside with a text editor to see that they mostly contain references to other files

Tools

  • Visual Studio Express is optimised for writing C#/VB code
  • Expression Blend is a design tool that is optimised for generating XAML
  • There is some support for designing XAML pages in Visual Studio and editing C#/VB code in Blend but the experience is better using both tools for their intended tasks
  • Editing XAML code directly is a better experience in Visual Studio
  • The main difference between the free Express edition of Visual Studio and the full version is that you cannot install plugins in the Express edition
  • The SDK includes a Windows Phone Emulator which is a Virtual Machine running the actual Windows Phone operating System
